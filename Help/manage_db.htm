Управление БД
 
     Окно управлением базы данных вызывается пунктом "Управление БД" из окна поиска или проводника. В данном окне возможно изменить практически любую информацию в базе данных.

     Для выбора базы данных, отличной от текущей, необходимо нажать на кнопку "Изменение файла БД". В открывшемся окне можно с помощью кнопки "Открыть" выбрать существующую БД или создано новую с помощью кнопки "Создать новую". Кнопка "Использовать БД по умолчанию" определяет, что открытая БД будет использоваться при следующей загрузке, а иначе банная БД будет использоваться только до перезагрузки приложения.

     Слева находится информация о текущей записи в базе данных: предпросмотр фотографии (взятый из базы данных), оценка, доступ, ID, Имя Файла\Путь к файлу\Комментарии\Ключевые слова, размер файла, разрешение исходной фотографии, дата фотографии (если существует), группы, к которым относится фотография, владелец и коллекция. Многие свойства можно изменить непосредственно тут, а некоторые можно в самой таблице:

Таблица базы данных содержит следующие основные поля:

ID (+) - Уникальный индетификационный номер фотографии
Name (Alpha) - Имя файла
FFileName (Memo) - Полный путь к файлу (указывается маленькими буквами для регистронезависимости поиска)
Comment (Memo) - Комментарии к фотографии
DateToAdd (Date) - Дата добавления файла в базу данных
IsDate (Bool) - Определяет, присутствует ли у фотографии конкретная дата
Owner (Alpha) - Владелец фотографии
Rating (Number) - Рейтинг фотографии
Thum (Blob) - Небольшое уменьшенное изображение фотографии (JPEG формат)
FileSize (Long Integer) - Размер файла
KeyWords (Memo) - Ключевые слова для поиска
Attr (Number) - Атрибуты записи (0 - нормальная запись, 1 - файла не существует, 2 - дубликат)
StrTh (Alpha) - Образ изображения (используется для индетефикации фотографий, поиска дубликатов и т.д.)
Collection (Alpha) - Имя коллекции, в которую входит данная фотография
Access (Long Integer) - Права доступа к фотографии (0 - фотография общего доступа, 1 - личная фотография)
Width (Long Integer) - Ширина фотографии
Height (Long Integer) - Высота фотографии
Rotated (Long Integer) - Пометка о положении фотографии (0 - нормальное положение, 1 - фотография будет повёрнута на 90 градусов, 2 - 180 градусов, 3 - 270 градусов)
Groups (Memo) - Содержит группы, к которым относится данная фотография
Include (Bool) - Фотография включается в базовый поиск
Links (Memo) - Ссылки к фотографии (формат ссылок - [тип_ссылки]{Имя}Значение;)
aTime (Date) - Время фотографии
IsTime (Bool) - присутствует ли время к фотографии

     Доступ к полю Groups напрямую запрещён. Изменения можно вносить через редактор групп.

     Генератор SQL-запросов позволяет выполнять некоторые действия ко многим фотографиям, позволяет установить какое-либо значение, либо удалить группу фотографий, заданную условием. Выполнение запроса начнётся после нажатия кнопки "Выполнить SQL"

     Дополнительные возможности:

"Упаковать БД" - Позволяет окончательно удалить стёртые записи (при обычном удалении они просто помечаются как удалённые).
Упаковка будет реализована при следующем запуске программы.

"Экспортировать" - Экспорт базы данных. В открывшемся окне необходимо выбрать путь к конечной базе данных, выбрать нужные опции экспорта и нажать на кнопку "Начать экспорт".

"Импортировать" - Импорт другой базы данных в текущую. Позволяет добавить новую информацию в базу данных и дополнить существующие записи из импортируемой базы данных.. В открывшемся окне нужно выбрать импортируемую базу данных, выбрать опции импорта и нажать на кнопку "Да", после чего будет произведёт импорт. Данная операция ведётся не полностью в автоматическом режиме, при добавлении информации программа может спрашивать о том, что делать с новыми данными (как, например, группами, отсутствующими в текущей БД, и т.д.).

"Перестроить IDEx" - обновляет уникальные индетефикаторы к файлом в БД, используйте при переходе от более ранней версии программы.

"Битые ссылки" - осуществляет поиск недействительных ссылок в БД

"Резервная копия" - Создание резервной копии базы данных. Резервная копия создаётся автоматически с интервалом около недели, но при необходимости можно выполнить резервное копирование в любое время.

"Очистка" - Просмотр и редактирование сервиса очистки и оптимизации БД.

"Резервные копии" - список резервных копий БД. Любую из них можно либо удалить (в контекстном меню) либо восстановить на место текущей БД. Текущая БД будет перед этим дополнительно резервно скопирована на случай неудачной операции. Восстановление происходит после перезапуска программы (необходимо для монопольного доступа к текущей БД) Все резервные копии находятся в списке "Резервные копии", из него можно восстановить любую резервную копию БД, выбрав в контекстном меню пункт "Восстановить". текущая БД при этом предварительно будет скопирована в резервное хранилище, восстановление произойдёт после перезагрузки программы.

Внимание: некорректное изменение информации может повлечь за собой неправильное интерпретирование данных программой.
